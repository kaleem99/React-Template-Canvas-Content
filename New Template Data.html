Introduction

Accessibility in web development is the designing and developing of websites and web applications in a way that makes them accessible to people with disabilities. This includes providing alternative ways for people with no visual contact, hearing or physical hands to access the content and functionality of a website.


Benefits

  • Increased reach: Making websites accessible allows your content to be used by a wider range of people including those with disabilities.

  • Improved user experience: Accessible websites are easier to navigate and use for people with disabilities, which can improve their overall experience and make it more likely for the users to engage more.

  • Improved website performance: Accessibility websites are designed to be lightweight and efficient, which can improve the sites performance and load times.

  • Future-proofing: Designing websites with Accessibility can help ensure that it remains functional and relevant for years to come even when technology and user needs evolve.


Proposed solution.

To Develop Interactive Templates using Accessibility, we proposed the following.

  • Buttons, images, videos and text elements should contain a tabIndex so that the content can be accessed through the keyboard.

  • Images should contain alt text so that users with visual impairments and screen readers can understand the image content when the screen reader describes it to them.

  • Use descriptive and meaningful alt text for images: Alt text is used to describe the content of images and provides an alternative to those who cannot see them. The alt text should accurately describe the content of the image.

Order of tabIndex was not done correctly, users that accessed the content through the use of keyboard had no control when it came to which elements should get accessed first
